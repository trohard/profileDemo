Test Server for Profile Database

 

<URL> to access site:

https://profiles-7xngsv.turbo360-vertex.com/

 

Field names for data (case matters!):

firstName

lastName

school

city

state

interests

 

API for accessing data:

Show all entries:

<URL>/api/profile

 

Show only entries that match a query string

<URL>/api/profile?<field name>=<string>

Example:

<URL>/api/profile?firstName=tom

 

Another way to show an entry based on id:

<URL>/api/profile/<id_no>

Example:

<URL>/api/profile/5cb4d21b3df79b231c499615

 

Update an entry based on id:

<URL>/api/profile/update?id=<id_no>&<field name>=<value>

Example:

<URL>/api/profile/update?id=5cb4d21b3df79b231c499615&interests=eating

It is possible to update more than one field at a time:

<URL>/api/profile/update?id=5cb4d21b3df79b231c499615&interests=eating&school=K-State

 

Remove an entry based on id:

<URL>/api/profile/remove?id=<id_no>

 

Add a new entry:

Using POST or GET method using a form, the action should be set to:

<URL>/api/profile/input

The form’s field names should match the database field names listed above

 

Using GET method on the URL:

<URL>/api/profile/input?<fieldname1>=<value1>&<fieldname2>=<value2>…

Example

<URL>/api/profile/input?firstName=Tom&lastName=Mertz&school=SCHS&city=Salina&state=KS&interests=Web
